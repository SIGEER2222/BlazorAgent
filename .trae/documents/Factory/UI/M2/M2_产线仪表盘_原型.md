M2 产线仪表盘 UI 原型

目标与路由
- 路由：`/factory/lines`
- 目标：按产线维度展示吞吐、能耗、占地与饥饿时间，支持动态增删机器与行级对比。

数据源与契约
- 产线列表：`GET /api/lines?factoryId=...` → `Line[]`
  - `Line`：`{ id, name, targetItemId, machines[], area{rows,cols} }`
- 行级指标：`GET /api/lines/{id}/metrics` → `LineMetrics`
  - `LineMetrics`：`{ throughput(items/s), energy(kWh), wip, starvationSeconds, inventoryDelta }`
- 设备增删：`POST /api/lines/{id}/machines`（`{ op:"add|remove", machine }`）

布局与交互
- 顶部工具栏：新建/删除产线、目标产物筛选、排序（吞吐/能耗/饥饿）。
- 列表视图：每条产线显示核心指标卡片与趋势小图（吞吐/能耗）。
- 详情抽屉：设备清单、占地统计、饥饿事件时间轴，支持增删设备与参数编辑。

指标说明
- 吞吐：`Produced[targetItemId] / ElapsedSeconds`（行级聚合）
- 能耗：行内设备 `recipe.powerKw` 累计（电力），燃料设备在燃料域统计（并在 UI 标注）。
- 饥饿时间：设备在输入不足时的累计时间，来源于引擎 `StarvationDetector`。

验收标准
- 增加设备后，吞吐与能耗出现期望变化（限电/燃料约束下可能非线性）。
- 输入带运力不足时，饥饿时间递增，吞吐下降。
- 产线排序与筛选功能工作正常，详情抽屉增删设备后指标随下一步进更新。

性能建议
- 指标拉取采用节流与增量；趋势图下采样与窗口化。
