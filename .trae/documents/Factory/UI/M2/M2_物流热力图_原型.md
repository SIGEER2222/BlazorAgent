M2 物流热力图 UI 原型

目标与路由
- 路由：`/factory/logistics`
- 目标：可视化带/臂/管的速率、容量与拥塞，定位瓶颈并支持优先级调整。

数据源与契约
- 网络数据：`GET /api/logistics/{factoryId}/network`
  - `nodes[]`：`{ id, type, position{row,col}, occupancy{rows,cols} }`
  - `edges[]`：`{ id, from, to, itemId, rate(items/s), capacity(items/s), buffer, latency }`
  - `congestion[]`：`{ edgeId, utilization, severity }`
- 优先级策略：`POST /api/logistics/{factoryId}/priority` → `{ edgeId, strategy }`
- 采样频率：默认 1s（可调），支持增量快照。

布局与交互
- 顶部工具栏：工厂选择、图层开关（带/臂/管）、颜色图例、采样频率选择。
- 画布层：
  - 节点按网格占位渲染；边按连接绘制，颜色映射 `utilization`（绿色→黄色→红色）。
  - 悬停显示速率/容量/缓冲；点击边弹出优先级策略设置（`PriorityFirst/RoundRobin`）。
- 侧边面板：拥塞列表（按 `severity` 排序），支持定位与修复建议入口。

颜色与图例
- 利用率 `u = rate / capacity`：
  - `u < 0.6` 绿色；`0.6 ≤ u < 0.85` 黄色；`u ≥ 0.85` 红色。
  - 缓冲高于阈值时显示虚线与警告图标。

验收标准
- 拥塞列表与画布颜色一致，点击定位准确。
- 调整策略后，目标边的 `rate` 或下游 `buffer` 出现预期变化（缓解/转移）。
- 在采样频率变化时，热力图稳定刷新且性能可控。

性能与可用性
- 大拓扑虚拟化：仅渲染视窗内节点与边，缩放时动态加载。
- 采样合并：对 200ms 内多次更新做合并，降低抖动。
