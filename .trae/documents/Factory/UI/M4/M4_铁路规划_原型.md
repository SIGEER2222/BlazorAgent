M4 铁路规划 UI 原型

目标与路由
- 路由：`/factory/rail`
- 目标：编辑站点与线路、规划车次与时刻表、避免冲突。

数据源与契约
- `POST /api/train/routes` → `RailRoute`
- `GET /api/train/schedule` → `{ trains[], stations[] }`

布局与交互
- 画布：站点节点与线路边；路径规划显示最短/最快方案。
- 时刻表视图：车次时间窗与装卸事件；冲突高亮与修复建议按钮。
- 操作：新增/删除站点与线路、生成路线与时刻表、调整车次参数。

验收标准
- 路线生成与时刻表无冲突或可修复；装卸规则执行正确。
