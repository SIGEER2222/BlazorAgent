M1 报表 UI 原型

目标与路由
- 路由：`/reports`（后续统一为 `/factory/reports`）
- 目标：展示累计指标与基础吞吐，验证 M1 仿真输出的正确性。

布局结构
- 概览卡片：`ElapsedSeconds`、`TotalEnergyKWh`
- 产出表：`Produced`（`itemId`、`amount`、`throughput`）
- 图表占位（可选）：吞吐折线（后续 M2 扩展）

计算逻辑
- `throughput = round(Produced[itemId] / ElapsedSeconds, 3)`（`ElapsedSeconds>0` 时）
- 能耗与时间均从仿真引擎读取（只读展示）

数据契约
- 直接从引擎 `SimulationEngine` 读取（现实现）
- 或通过 `GET /api/sim/metrics` 拉取快照（推荐用于前后端分离）

验收标准
- 在仿真运行后，`Produced` 列表非空且吞吐为正
- 能耗随时间累积且与步进批次一致
