M1 仿真控制 UI 原型

目标与路由
- 路由：`/factory`（或规划为 `/factory/sim/{id}`）
- 目标：提供仿真状态控制与指标面板，验证步进、能耗与吞吐的正确性。

功能模块
- 控制区：`Start`、`Pause`、`Step`、步长（`Step Seconds`）
- 指标区：`ElapsedSeconds`、`TotalEnergyKWh`、`Produced` 列表
- 库存区：`Inventory` 列表

交互流程
1. 初始化：加载 `items/recipes`，若缺少原料，调用 `Seed("iron-ore", 10)`
2. Start：设置 `Running=true`，启动 `PeriodicTimer(250ms)`；循环执行 `Step(stepSeconds)`
3. Pause：设置 `Running=false`，取消计时器
4. Step：单次推进，引擎更新库存/产出/能耗，UI 刷新

数据契约
- `GET /api/items`、`GET /api/recipes`（或直接注入仓库）
- `GET /api/sim/metrics`：`{ energyKWh, inventory{itemId→amount}, produced{itemId→amount} }`

错误与边界状态
- 原料不足：产出停止，能耗仅在完成批次时累计
- 能源限额：`PowerDomainKw` 限制造成有效秒数缩放；吞吐按比例下降
- 时间不足：`effectiveSeconds < recipe.timeSeconds` 不产生批次

验收标准
- 多次 `Step(1.0)` 后，`iron-plate` 数量递增；吞吐≈`Produced[iron-plate]/ElapsedSeconds`
- 调整 `PowerDomainKw` 时吞吐按比例变化
