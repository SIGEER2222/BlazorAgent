M1 设计器 UI 原型

目标与路由
- 路由：`/factory`（后续统一为 `/factory/designer`）
- 目标：提供网格画布与基本仿真控制，支持库存与产出可视化，作为蓝图编辑器的雏形。

布局结构
- 顶部控制栏：`Start`、`Pause`、`Step`、`Step Seconds` 滑条（0.1–2.0）
- 中部画布：12×8 网格，设备占位块（Assembler/Belt/Output）示例
- 底部面板：
  - `Inventory` 表：`itemId`、`amount`
  - `Metrics`：`ElapsedSeconds`、`TotalEnergyKWh`、`Produced` 表（`itemId`、`amount`）

数据绑定
- 物料与配方：`GET /api/items`、`GET /api/recipes`
- 仿真指标：`GET /api/sim/metrics`（`energyKWh`、`inventory`、`produced`）
- 组件注入（现实现）：`SimulationEngine`、`ICatalogRepository`

关键交互
- Start：启动周期计时器（250ms），每 tick 执行 `Step(stepSeconds)` 并刷新 UI
- Pause：停止计时器与仿真
- Step：执行一次步进并刷新 UI
- Step Seconds：滑条调整每次步进的秒数，影响吞吐与能耗累计速率

占位与约束（M1 范围）
- 设备示意占位：在网格画布以块状渲染，不进行碰撞检测（M2 引入占位校验）
- 连线与约束提示：M1 仅静态摆放，不进行连线校验（M2 引入）

UI 元素清单
- 控件：`MudButton(Start/Pause/Step)`、`MudSlider(Step Seconds)`、`MudTable(Inventory/Produced)`、`MudText(Metrics)`
- 组件：`FactoryCanvas`（网格画布原型）

验收标准
- 点击 `Start` 后，`iron-plate` 在库存与产出表中随时间增加
- `Energy` 与 `ElapsedSeconds` 随步进递增；`Step Seconds` 改变后，吞吐变化符合线性预期
