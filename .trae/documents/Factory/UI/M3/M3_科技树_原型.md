M3 科技树 UI 原型

目标与路由
- 路由：`/factory/research`
- 目标：展示科技树结构、管理研究队列、显示研究进度与解锁效果。

数据源与契约
- `GET /api/research/tree` → `ResearchNode[]`
  - `ResearchNode`：`{ id, name, cost{itemId→amount}, unlocks{recipes[], machines[], modules[]}, deps[] }`
- `POST /api/research/queue` → `{ queue:[nodeId], progress:{ nodeId:0..1 } }`
  - 请求：`{ add:[nodeId], remove:[nodeId] }`

布局与交互
- 左侧树图：节点显示名称与状态（未解锁/排队/进行中/完成），依赖以连线呈现。
- 右侧面板：队列列表、进度条、解锁预览（新增配方/设备/模块）。
- 操作：点击节点添加至队列；拖拽调整队列顺序；移除节点；查看解锁影响。

状态与提示
- 依赖未满足时，节点显示禁用状态与缺失依赖提示。
- 研究进行中显示预计完成时间（基于 `researchRate` 与 `cost`）。

验收标准
- 队列增删与排序正确、进度显示与树状态一致。
- 解锁后新配方/设备在规划页面可用。
