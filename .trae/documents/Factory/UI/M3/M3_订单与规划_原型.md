M3 订单与规划 UI 原型

目标与路由
- 路由：`/factory/planning`
- 目标：基于订单生成产线方案，展示差距与瓶颈，提供优化建议。

数据源与契约
- `POST /api/orders` → `{ id, status }`
- `GET /api/orders/{id}` → `Order`
- `POST /api/plans/generate` → `Plan`
- `GET /api/plans/{id}/bottlenecks` → `Bottleneck[]`
- `GET /api/plans/{id}/suggestions` → `Suggestion[]`

布局与交互
- 顶部区：订单选择、目标节拍/约束输入（电力/燃料/带速）、优化目标选择。
- 中部方案视图：行级方案卡片（产线→目标产物→设备清单→预估吞吐/能耗）。
- 右侧差距与瓶颈：`gaps` 列表与瓶颈条目，点击显示建议与预计收益。
- 操作：应用建议（升级设备/增设分流/调整优先级/提升电力域/增加燃料供给）。

成本与模式（新增）
- 成本面板：显示 `holding/shortage/energy` 分项与总成本；切换 `objective`。
- 模式切换：`pull|push`，影响规划约束（交期 vs 安全库存）与建议排序。
- 仓储联动：展示 `Warehouse` 库存与 `safetyStock`；支持策略参数（再订货点/看板）。

周期与交期（新增）
- 指标区：显示 `ctSeconds/ltSeconds/promiseDate/due` 与 `dueRisk`；风险为中高时高亮并给出加急建议入口。
- 计划日程：提供简化日程视图，显示关键工位的等待与加工时间。

产能与良率（新增）
- 行级卡片：显示 `capacity_nominal/effectiveYield/capacity_effective` 与 `scrapRate`。
- 良率编辑（可选）：在建议面板中支持设备升级或质量模块调整，预估 `yieldRate` 变化对产出影响。

验收标准
- 生成方案耗时可控（中等规模 `< 2s`），瓶颈与差距展示正确。
- 应用建议后方案更新并显示吞吐改善或能耗权衡。
- 切换拉/推模式与成本目标时，方案的成本分解与交期/服务水平指标符合预期。
 - 周期/交期与产能/良率指标在方案与仿真观测之间一致；风险提示与加急建议有效。
