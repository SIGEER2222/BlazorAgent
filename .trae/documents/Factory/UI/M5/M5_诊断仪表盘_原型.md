M5 诊断仪表盘 UI 原型

目标与路由
- 路由：`/factory/diagnostics`
- 目标：展示性能与稳定性指标，支持故障注入与恢复观察。

数据源与契约
- `GET /api/diagnostics/metrics` → 指标快照
- `POST /api/diagnostics/chaos` → 启动故障注入

布局与交互
- 指标卡片：TPS、延迟、CPU、内存、GC 暂停、丢 Tick。
- 趋势图：实时曲线（节流与下采样）。
- 故障注入面板：模式选择（丢包/限电/延迟/燃料中断）、目标范围与持续时间。

验收标准
- 指标与故障状态一致；恢复时间与稳态识别正确。
